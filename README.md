# Draeyes Remote C6 完整版遥控器

Draeyes Remote C6 是 Draeyes 兽装可动眼系列的完整版手持遥控器固件与硬件资料仓库。该版本面向 ESP32-C6 主控硬件，使用摇杆、肩键、同步拨动开关和可选 VEML7700 环境光传感器，通过纯 ESP-NOW 向接收端发送低延迟控制数据。

完整版遥控器适合正式使用场景：支持手动视线控制、全自动仿生眼动、自动眨眼、左右眼皮手动控制、摇杆锁定、摇杆零点校准、接收端配对，以及基于环境光的亮度或瞳孔响应控制。

早期设计中预留过 WiFi AP 和 MQTT 辅助功能，当前公开版本已经移除这些未使用模块，以减少固件体积、依赖库和运行时内存占用。

## 硬件配置

| 项目 | 说明 |
| --- | --- |
| 主控 | ESP32-C6 |
| 无线协议 | ESP-NOW |
| 主要输入 | 二轴模拟摇杆 |
| 按键 | R1、R2、LB、RB |
| 拨动开关 | BK，左右眼皮同步 |
| 光照传感器 | VEML7700，I2C |
| 参数保存 | EEPROM 模拟，用于保存摇杆零点和亮度补偿 |

| 信号 | GPIO |
| --- | --- |
| LB | 20 |
| RB | 18 |
| 摇杆 X | 5 |

| 摇杆按键 | 21 |

## Arduino 依赖

请在 Arduino IDE 或 PlatformIO 中安装：

| 库 | 用途 |
| --- | --- |
| ArduinoJson | 生成 ESP-NOW JSON 数据包 |
| Adafruit VEML7700 | 读取环境光传感器 |

同时需要安装支持 ESP32-C6 的 ESP32 Arduino Core。

## 编译与烧录

1. 用 Arduino IDE 打开 `Draeyes-Remote-C6.ino`。
2. 选择 ESP32-C6 对应开发板。
3. 安装上面列出的依赖库。
4. 使用 USB 连接遥控器。
5. 编译并上传固件。
6. 打开串口监视器，波特率设为 `115200`，查看启动、配对、校准和调光信息。

## 通信数据格式

控制器通过 ESP-NOW 发送 JSON 数据。

常规控制数据：

```json
{
  "req": "controller",
  "data": {
    "light": 1.0,
    "j1PotX": 127,
    "j1PotY": 127,
    "bkl": 1.0,
    "bkr": 1.0
  }
}
```

配对请求：

```json
{ "req": "pairing" }
```

眼皮样式切换请求：

```json
{ "req": "change" }
```

### 字段范围

| 字段 | 范围 | 含义 |
| --- | --- | --- |
| `j1PotX` | `0..255` | 眼球水平位置 |
| `j1PotY` | `0..255` | 眼球垂直位置 |
| `bkl` | `0.0..1.0` | 左眼皮开合度 |
| `bkr` | `0.0..1.0` | 右眼皮开合度 |
| `light` | `0.02..1.0` | 亮度或瞳孔光照控制值 |

## 按键布局

```text
          [ 摇杆 ]

      [ R1 ]       [ R2 ]

右侧边缘：
      [ LB ]  左眼皮 / 亮度增加
      [ BK ]  左右眼皮同步
      [ RB ]  右眼皮 / 亮度减少
```

## 常规操作

| 操作 | 功能 |
| --- | --- |
| 摇杆 | 控制眼球视线方向 |
| LB | 控制左眼皮闭合或眨眼 |
| RB | 控制右眼皮闭合或眨眼 |
| BK | 开启时右眼皮跟随左眼皮，实现双眼同步 |
| R1 连按 3 次 | 循环切换运动模式 |
| R2 连按 3 次 | 发送眼皮样式切换请求 |
| R2 长按 3 秒 | 锁定或解锁摇杆 |
| R1 + R2 长按 3 秒 | 发送接收端配对请求 |
| R1 + R2 + LB + RB 长按 3 秒 | 校准摇杆中心 |

## 运动模式

通过 R1 连按 3 次循环切换。

| 模式 | 视线控制 | 眼皮控制 |
| --- | --- | --- |
| 手动模式 | 摇杆控制 X/Y | LB/RB 手动控制左右眼皮 |
| 全自动模式 | 自动仿生眼动 | 自动眨眼 |
| 自动眨眼模式 | 摇杆控制 X/Y | 自动眨眼 |
| 锁定加自动眨眼模式 | 视线居中或锁定 | 自动眨眼 |

## 配对说明

1. 打开接收端设备。
2. 打开遥控器。
3. 同时按住 `R1 + R2` 约 3 秒。
4. 遥控器发送 `{ "req": "pairing" }`。
5. 接收端开始响应控制后即可松开按键。

当前固件使用 ESP-NOW 广播地址发送配对请求，接收端需要实现对应的配对处理逻辑。

## 摇杆校准

当松开摇杆后眼球仍然偏移时，需要执行摇杆零点校准。

1. 保证摇杆处于物理中心位置。
2. 同时按住 `R1 + R2 + LB + RB` 约 3 秒。
3. 串口输出 `Writing in EEPROM...` 和 `Done...` 后校准完成。
4. 松开所有按键。

校准后的摇杆中心值会保存到 EEPROM，下次开机继续生效。

## 摇杆锁定

在手动模式或自动眨眼模式下：

1. 长按 `R2` 约 3 秒。
2. 遥控器切换摇杆锁定状态。
3. 锁定后发送的 X/Y 为中心值。
4. 再次长按 `R2` 约 3 秒解除锁定。

## 环境光调光

VEML7700 光照传感器是可选硬件。固件启动时会自动检测传感器：

- 检测到传感器：启用调光配置模式。
- 未检测到传感器：禁用调光配置模式，并固定发送 `light = 1.0`。

### 进入与切换调光模式

长按 `R1` 约 3 秒，按下面顺序循环：

```text
主模式 -> 调光子模式 1 -> 调光子模式 2 -> 主模式
```

离开调光子模式时，亮度补偿参数会保存到 EEPROM。

### 调光子模式 1：跟随设置

| 操作 | 功能 |
| --- | --- |
| R2 单击 | 开启或关闭环境光跟随 |
| LB 单击 | 亮度补偿增加 `0.05` |
| RB 单击 | 亮度补偿减少 `0.05` |
| R1 长按 3 秒 | 保存并进入调光子模式 2 |

关闭环境光跟随后，遥控器发送 `light = 1.0`。

### 调光子模式 2：采样锁定

| 操作 | 功能 |
| --- | --- |
| R2 单击 | 采样当前环境光并锁定该亮度值 |
| LB 单击 | 亮度补偿增加 `0.05` |
| RB 单击 | 亮度补偿减少 `0.05` |
| R1 长按 3 秒 | 保存并退出到主模式 |

该模式适合现场灯光复杂、希望固定亮度或瞳孔响应的场景。

## 常见问题

### 长按 R1 不能进入调光模式

检查 VEML7700 的 SDA/SCL 接线，并重启遥控器。固件只有在启动时检测到传感器，才会启用调光模式。

### 摇杆回中后眼球仍偏移

执行 `R1 + R2 + LB + RB` 长按 3 秒的摇杆校准。

### 接收端没有响应

确认接收端已开机，并重新执行 `R1 + R2` 配对。还需要确认双方使用同一个 ESP-NOW 信道。

### 编译时出现 `wifi_tx_info_t` 相关错误

请升级 ESP32 Arduino Core。ESP32-C6 需要较新的 ESP-NOW 回调类型支持。
